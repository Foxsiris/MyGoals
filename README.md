# Рост — привычки и цели

Веб-приложение для личной продуктивности: ежедневные **привычки** (серии + тепловая карта) и большие **цели** с декомпозицией на этапы и шаги. Привычки питают цели; прогресс целей считается автоматически. Плюс **матрица Эйзенхауэра** и **статистика** по привычкам.

Сделано на **React 19 + Vite 8**, все данные хранятся в **Supabase** (PostgreSQL). Тёплая светлая тема, два акцента (малиновый — цели, зелёный — привычки), шрифты Manrope + Golos Text. Адаптивно: десктоп (сайдбар) и мобильная раскладка (нижняя таб-навигация).

## Экраны

- **Дашборд** — что делать прямо сейчас: привычки на сегодня, ближайшие шаги к целям, плитки серии/прогресса/недели.
- **Привычки** — отметка выполнения, текущая и лучшая серия, % за период, тепловая карта в стиле GitHub.
- **Цели** — карточки с прогресс-баром, индикаторами этапов и счётчиком привязанных привычек.
- **Декомпозиция цели** — дорожная карта: кольцо прогресса, мотивация «Зачем», привычки, питающие цель, колонки этапов с инлайн-добавлением шагов.
- **Матрица Эйзенхауэра** — распределение задач по срочности × важности (drag-and-drop).
- **Статистика** — KPI, динамика по неделям, сильные дни недели, проценты по привычкам.
- **Создание/редактирование** целей и привычек.

## Стек

- [React 19](https://react.dev/) + [Vite 8](https://vite.dev/)
- [Supabase](https://supabase.com/) — БД и API (клиент `@supabase/supabase-js`)

## Модель данных (Supabase)

| Таблица | Назначение |
| --- | --- |
| `goals` | цели (название, мотивация, срок, заметка) |
| `stages` | этапы цели |
| `steps` | шаги этапа (выполнено/нет) |
| `habits` | привычки (частота, иконка, напоминание) |
| `habit_goals` | связь привычка ↔ цель |
| `habit_logs` | отметки выполнения по дням (серии, тепловая карта, статистика) |
| `matrix_tasks` | задачи матрицы Эйзенхауэра |

Серии, рекорды, проценты и тепловая карта вычисляются на клиенте из реальных `habit_logs`.

## Запуск

1. Установить зависимости:
   ```bash
   npm install
   ```
2. Указать доступ к Supabase в `.env` (значения уже зашиты как fallback в `src/lib/supabase.js`, так что приложение работает и без файла):
   ```
   VITE_SUPABASE_URL=https://<project>.supabase.co
   VITE_SUPABASE_ANON_KEY=<publishable-key>
   ```
3. Запустить:
   ```bash
   npm run dev      # http://localhost:5173
   npm run build    # production-сборка в dist/
   npm run preview  # просмотр сборки
   ```

> Приложение настроено как личное (без авторизации): одна общая база, доступ — публичным ключом, защита — через RLS. Чтобы сделать многопользовательским, добавь Supabase Auth и привяжи строки к `user_id`.
